# هندسة المنتج — إعادة بناء ذكية

## إدخال المنتجات أول مرة:
1. التاجر يختار نوع بيزنسه عند التسجيل
2. النظام يعرض: "عندنا قوالب جاهزة لـ [نوعك] — تبي نضيفها؟"
3. بضغطة واحدة = 5-15 منتج/خدمة جاهزة بأسعار مقترحة
4. التاجر يعدّل اللي يبي ويحذف اللي ما يبي
5. النتيجة: من 0 إلى جاهز في دقيقتين

## فورم المنتج/الخدمة الذكي:
### المستوى 1 (يظهر دائماً — إلزامي):
- اسم المنتج/الخدمة
- السعر
- حفظ ←

### المستوى 2 (يظهر لو ضغط "تفاصيل إضافية"):
- الوصف
- الصورة
- التصنيف
- المدة (للخدمات)
- الكمية (للمنتجات)

### المستوى 3 (يظهر لو ضغط "خيارات متقدمة"):
- الوصفة/المكونات (ربط بالمخزون)
- IMEI/Serial (للإلكترونيات)
- تاريخ الانتهاء
- الضمان
- variants (ألوان/مقاسات)
- خدمات إضافية مرتبطة

## حذف الأنظمة المكررة:
- إذا فيه booking engine + reservation engine = وحّدهم
- إذا فيه payment handler في 3 أماكن = مكان واحد
- إذا فيه navigation registry في ملفين = ملف واحد

## حذف الصفحات الميتة:
- أي صفحة ما فيها محتوى حقيقي = تنحذف
- أي route ما يوصل لصفحة = ينحذف
- أي component ما يستخدمه أحد = ينحذف
