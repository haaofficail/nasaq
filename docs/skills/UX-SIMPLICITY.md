# قواعد التبسيط والذكاء — نسق

## القاعدة #1: خطوة أو خطوتين لكل إجراء
- إضافة منتج: اسم + سعر + حفظ (خطوة واحدة)
- التفاصيل المعقدة (وصفة، مكونات، ضمان) = قسم "خيارات متقدمة" مطوي افتراضياً
- التاجر يفتحه لو يبي — مو مجبور

## القاعدة #2: قوالب منتجات جاهزة لكل نوع بيزنس
- أول ما التاجر يسجّل ويختار نوعه = يظهر له: "تبي نضيف منتجات جاهزة؟"
- صالون: قص شعر، صبغة، استشوار، بروتين، ميش، مكياج، حناء — بأسعار مقترحة
- مقهى: إسبريسو، لاتيه، كابتشينو، V60، موكا، شاي — بأسعار مقترحة
- مطعم: برقر، بيتزا، سلطة، عصير — بأسعار مقترحة
- ورد: باقة ورد أحمر، باقة مختلطة، تنسيق طاولة — بأسعار مقترحة
- تأجير: خيمة صغيرة، خيمة كبيرة، كوشة، طاولات — بأسعار مقترحة
- التاجر يعدّل الأسماء والأسعار أو يحذف اللي ما يبيه

## القاعدة #3: الأسماء الأوضح والأسهل
- "إدارة المخزون" → "المستودع"
- "إدارة الموارد البشرية" → "الموظفين"
- "التقارير المالية" → "التقارير"
- "إدارة العملاء" → "العملاء"
- "نقطة البيع" → "الكاشير"
- كل اسم يكون كلمة واحدة أو كلمتين — بدون "إدارة" و"نظام"

## القاعدة #4: ترتيب الخيارات حسب الأولوية
- أول دخول للداشبورد: الأكثر استخداماً فوق
  1. الحجوزات/الطلبات (الأهم)
  2. الكاشير (لو فيه بيع مباشر)
  3. العملاء
  4. المنتجات/الخدمات
  5. المستودع
  6. الفواتير
  7. التقارير
  8. الإعدادات (آخر شيء)

## القاعدة #5: لا تكرار — لا أنظمة ميتة
- ما فيه نسختين من نفس الوظيفة (مثلاً: booking engine + reservation engine = واحد بس)
- أي صفحة ما تستخدم = تنحذف
- أي endpoint ما له صفحة = ينحذف أو يتربط
- أي جدول DB فاضي ومو مستخدم = يتوثّق للحذف

## القاعدة #6: النظام يكون ذكي
- إذا التاجر ما أضاف منتجات بعد → يظهر: "ابدأ بإضافة أول منتج" مع زر واضح
- إذا المخزون قرب يخلص → إشعار تلقائي
- إذا عميل ما جا من شهر → اقتراح: "أرسل رسالة تذكير؟"
- إذا يوم فيه حجوزات كثير → تنبيه: "يومك مزدحم — تأكد من الجاهزية"
- الداشبورد يعرض أرقام اليوم: حجوزات اليوم + إيراد اليوم + عملاء جدد
